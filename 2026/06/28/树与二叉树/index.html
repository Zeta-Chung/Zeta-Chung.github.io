数据结构-树与二叉树

发表于 2026-06-28 更新于 2026-06-29

数据结构中，树与二叉树的学习笔记，包括树与二叉树的定义、存储结构、基本操作等内容，用C语言实现。

树

树的基本定义

树的基本概念

树是n个结点的有限集合T，当n=0时称为空树，当n大于0时满足以下条件：

  1. 树中至少有一个结点，该结点称为根结点，没有直接前驱，但有零个或多个直接后继。

  2. 其余n-1个结点可以划分成m个互不相交的子集，每个子集都是一棵树，这些子树互为子树。

  3. 每棵子树的根节点有且仅有一个直接前驱，但有零个或多个直接后继。

树的图解表示法：

  1. 倒置树结构

  2. 文氏图表示法

  3. 广义表形式（嵌套括号表示法）：如：(1,(2,3))，表示一个树，根结点为1，有2个子树，第一个子树根结点为2，第二个子树根结点为3。

  4. 凹入表示法：用位置的缩进表示其层次。

树的相关术语

  1. 结点：包含一个数据元素以及若干指向其他节点的分支信息

  2. 节点的度：一个结点所拥有的分支数（子树的个数）

与图论中的度区分：图论中的度需考虑入度和出度，而数据结构中的度只考虑出度（除根结点外入度恒为1，用来描述结点特征没有意义）。

  1. 叶结点：度为0的结点，也叫终端结点。

  2. 分支结点：度大于0的结点，也叫非终端结点。

  3. 结点的层次（深度）：根结点为第0层，其直接后继为第1层，依次类推。

  4. 结点的层序编号：从1开始，根结点为1，其直接后继为2，依次类推。

  5. 树的度：树中结点的最大度数。

  6. 树的高度（深度）：树中结点的最大层次。

  7. 有序树：树中各子树Ti之间有先后次序。

  8. 森林：m棵互不相交的树的集合,m>=0。

  9. 同构：两棵树对结点适当重命名，便可完全相等。（旋转、镜像、交换、交换位置、交换子树等操作后，两棵树相等）

  10. 孩子结点：树中某个结点的直接后继。

  11. 双亲结点：树中某个结点的直接前驱。

  12. 兄弟结点：同一双亲结点的孩子结点之间互称兄弟结点。

  13. 堂兄弟结点：父亲是兄弟或堂兄弟关系的结点。

  14. 祖先结点：树中某个结点到根结点的路径上的所有结点。

  15. 后代结点：树中某个结点到叶子结点的所有路径上的所有结点。

  16. 前辈：层号比某结点小的结点，都是该结点的前辈。

  17. 后辈：层号比某结点大的结点，都是该结点的后辈。

树的抽象数据类型

  • 数据对象：一个集合D，D中所有元素具有相同的属性。

  • 结构关系：若D为空集，则为空树。若D仅含一个数据元素，则关系R为空集，否则R={H}，H是如下二元关系的集合：

    1. D中存在唯一的根结点root,它在H下没有前驱。

    2. 除root外，D中每个结点有且只有一个直接前驱。

  • 基本操作：

    1. 初始化树
    2. 销毁树。
    3. 创建树
    4. 判断树是否为空树

    6. 获取某个结点x的双亲结点
    7. 获取某个结点x的第一个子结点
    8. 获取某个结点x的下一个兄弟结点
    9. 插入子树（将子树child插入Tree中作为结点p的子树）

    11. 按次序访问树中的结点

性质

  1. 二叉树的第i层至多有 2i−12^{i-1}2i−1个结点。

  2. 深度为k的二叉树至多有 2k−12^k-12k−1个结点。

  3. 二叉树中结点数等于其度数之和加1。（或：二叉树的终端结点数等于其度数为2的结点数加一）

  4. 两种特殊的二叉树：

    1. 满二叉树：深度为k且结点数为 2k−12^k-12k−1的二叉树。

      • 满二叉树中，第i层结点数为 2i−12^{i-1}2i−1。

      • 满二叉树的顺序表示：从根开始，按层间从上到下，层内从左到右的顺序逐层编号。

    2. 完全二叉树：一棵深度为k的二叉树，前k−1层都是满的（结点数达到最大值），且最后一层的结点从左到右连续排列，中间不能有空缺。

      • 叶结点只可能出现在最后两层。

      • 度为1的结点个数为0或1

      • 具有N个结点的完全二叉树，其深度为 [log⁡2N]+1[\log_2N]+1[log2​N]+1）。

      • 对于具有N个结点的完全二叉树，对其顺序编号，则对任意序号为i的结点有：

        1. 若i=1,则i为根节点；若i>1，则i的父节点为 i2\frac{i}{2}2i​(向下取整)。

        2. 若2i>n,则i无左子结点；若2i<=n,则i有左孩子结点且编号为2i；

        3. 若2i>n,则i无右子结点；若2i+1<=n,则i有右孩子结点且编号为2i+1。

        • 注：编号从1开始，即使某个位置上没有结点，也要保留它的编号。（位置可以空，编号不能省）

    完全二叉树不一定是满二叉树，满二叉树一定是完全二叉树。

  3. 非递归实现遍历（调用栈操作函数）

    • 算法思想：

      1. 入栈：只要当前结点 p 不为空，就把它压入栈中，然后立刻转向其左孩子（p = p->LChild）。

      2. 动作一直持续到遇到 NULL（即最左下角的空孩子）。此时，栈中存放的是从根结点到最左下角结点的完整路径。

      3. 出栈：当 p 为空时，说明当前这条左分支走到了头。此时只窥探（GetTop）栈顶结点，并不急着弹出。

      4. 判断能否访问栈顶结点：

        1. 可以访问：如果栈顶结点没有右孩子（p->RChild == NULL），或者它的右孩子刚刚被访问过（p->RChild == q）。这说明左右子树都已处理完毕，可以访问它了。于是：弹出（Pop）、访问（visit）、用 q 记录它、将 p 置为 NULL（以便继续回溯）。

        2. 暂不能访问：如果栈顶结点有右孩子且没有被访问过，说明右子树还没处理。此时不弹出，而是把 p 指向右孩子（p = p->RChild），然后进入下一轮大循环。下一轮会立刻执行“一路向左”，把右子树的左链全部压栈。

      5. 循环结束：当栈空且 p 为 NULL 时，所有结点访问完毕。

线索二叉树

定义

  • 线索二叉树是对二叉树的结点增加两个指针域，分别用于指向该结点的前驱和后继结点，以实现中序遍历的线索化。

  • 节省内存的方法：把已经定义的空指针域存储为线索。

    • 例如：一个结点的左孩子不存在时，利用其空闲的左指针存储前驱内容；右孩子不存在时，用右指针存储后继内容。

    • 比起另外定义两个指针来存储前驱和后继，这种方法只定义了两个标志位，且利用起原本为NULL的指针，所以节省了内存。

      • 普通二叉树中序遍历，需要借助递归（系统栈）或自己建栈，时间复杂度 O(n)，空间复杂度 O(树高)。

      • 线索二叉树中序遍历，只要顺着 Rtag == 1 的线索一路找后继，不需要栈，空间复杂度降为 O(1)。

  • 定义方法

  • 前驱/后继同左/右孩子的区别

    对比维度 左/右孩子（物理结构 前驱/后继（遍历序列）
    本质 静态的父子关系（边） 动态的访问顺序（线性序列中的相邻关系）
    决定因素 由树的形态（插入方式）决定 由遍历算法（前/中/后序）决定
    是否唯一 是确定的（左孩子只有一个） 不确定（同一棵树，中序、先序、后序的前驱/后继完全不同）
    存储依赖 必须存储在结点的指针域中 不占额外存储（靠遍历过程获得，或利用空链域存线索）
    与空指针的关系 若没有，则该域为 NULL 中序首结点无前驱，末结点无后继（但可指向头结点）
    典型用途 构成树的结构本身 线性化树，方便查找某个结点的“下一个访问对象”

    “左右孩子”是针对于“这个结点自己”来说的；而“前驱后继”是针对“整个树的遍历结果”来说的。
    线索化只是"借用"了闲置的左右指针空间，让它们临时（或永久）去指向前驱/后继。这不意味着"左孩子"就等于"前驱"。

由遍历序列确定二叉树

组合方式 结论
先序 + 中序 唯一确定
后序 + 中序 唯一确定
先序 + 后序 不能唯一确定（除非是满二叉树）

原因：

  1. 先序提供根结点的位置（第一个元素一定是根），中序提供左右子树的分界（根左边是左子树，右边是右子树）。

  2. 后序提供根结点的位置（最后一个元素一定是根），中序同样提供左右子树的分界。

  3. 先序和后序都只能确定“谁是根”，但无法区分“谁是左孩子，谁是右孩子”。当某个结点只有一个孩子时，先序+后序无法唯一确定该孩子是左还是右。

哈夫曼树

哈夫曼树的基本概念

  1. 前置基本概念：

    1. 路径：从一个结点到另一个结点的分支序列

    2. 路径长度（PL）：路径上分支的数目

    3. 树的路径长度： 根节点到树中所有节点的路径长度之和

    4. 结点的权：给结点赋予的一个具有某种实际意义的实数

    5. 带权路径长度：树根到某一结点的路径长度乘以该结点的权

    6. 树的带权路径长度：树中所有带权路径长度之和。

      通常记为： WPL = ∑i=1nwi∗li\sum_{i=1}^{n}w_i*l_i∑i=1n​wi​∗li​

  2. 定义：用n个带权叶结点构成的所有二叉树中带权路径长度最短的树，又称为最优二叉树。

  3. 构建步骤

    1. 初始化：用给定的n个权值构建n棵二叉树，构成一片森林，其中每棵二叉树只有一个带权叶结点，左右子树为空。

    2. 合并：在已有的n棵二叉树中，选择两棵根结点的权值最小的树作为左右子树，合并为一棵新的二叉树，新二叉树的权值为左右子树权值之和。

    3. 删除与加入：删除被合并的两棵二叉树，把它们新构成的二叉树加入森林。

    4. 重复2，3，直到只剩下一棵树为止。

  4. 哈夫曼树的类型定义

    1. 存储结构：n个叶子的哈夫曼树一共有(2n-1)个结点，所以用一个大小为2n-1的一维数组存放各个结点，每个结点还包含其双亲和孩子结点的信息，构成一个静态三叉链表

      • 静态：已知叶子数目则总结点数固定，可以用数组来存储。

      • 三叉：每个结点包含双亲、左孩子和右孩子的信息。

      • 为方便实现，将叶结点集中存储在1n个位置，非叶结点集中存储在n+12n的位置。

哈夫曼编码

基本概念

  1. 为了缩短数据文件长度，可采用不定长编码，给使用频率高的字符编码较短，给使用频率低的字符编码较长。

  2. 前缀编码：在一个编码系统中，任一编码都不是其他编码的前缀。

  3. 哈夫曼编码：给定n个权值，构造一棵哈夫曼树，从根结点到叶结点的路径上的分支编号为0和1，从左到右，则这条路径的编码就是这条路径上的分支编号所组成的序列。

  4. 哈夫曼编码的性质：

    1. 哈夫曼编码是前缀编码

    2. 哈夫曼编码是最优前缀编码
