stm32时钟树模块的学习笔记（基于铁头山羊STM32教程，含自主探索内容）

时钟和时钟树的概念

时钟：持续振荡、高低电平交替变化的方波信号。

时钟树： 不同外设（如 GPIO、USART、定时器）可能工作在不同的时钟频率下，因此需要一套 时钟分配和倍频/分频网络，这就是时钟树。

如图，STM32F103C8T6里的时钟树结构，其中背景带颜色表示可开关，带绿色表示默认开启

分频器、锁相环和复用器

分频器：对频率做除法。

锁相环（phase lock loop,PLL）：对频率做乘法。

复用器：对频率做选择。

树根

四个时钟源

内 外
低 LSI(Low Speed Internal) LSE(Low Speed External)
高 HSI(High Speed Internal) HSE(High Speed External)

  • 高低之分：左边的大树（高速树）上时钟频率较高，通常为几十MHz；
    右边的小树（低速树）上时钟频率较低，通常为几十KHz。

  • 内外之分：时钟源位于单片机内部（不需要外接）或外部（需要外接）。

    外接原因：内部时钟源的精度不高，对精度有要求时需要外接。

HSE 高 等于HSE频率

树枝

  • AHB总线分出APB1和APB2两条总线，两条总线上各自挂载不同外设，AHB总线上挂载DMA1与DMA2外设。

  • 每一根树枝都有一个分频器，AHB分频器产生HCLK信号后，APB1或APB2分频器可以继续对其分频。

时钟树的编程

时钟树的初始状态

头文件含SystemInit的状态:

  • SYSCLK：来自HSE，经分频器和锁相环，72MHz。
  • AHB和APB2：分频系数为1，72MHz.
  • APB1：分频系数为2，36MHz.

    注：SystemInit函数配置了时钟树的初始状态，通常在启动文件中被调用，确保系统时钟在上电后就能稳定运行。

头文件去掉SystemInit的状态()：

  • SYSCLK：来自HSI，8MHz。
  • APB2、AHB、APB1分频器：分频系数均为1，8MHz。

    注：此后的笔记均在去掉SystemInit的状态下进行，除非特别说明。

时钟树的编程接口

  1. void RCC_DeInit(void)

    • 功能：复位所有 RCC 寄存器到默认值（系统时钟恢复为 HSI，所有外设时钟关闭）。

    • 参数：无。

  2. void RCC_HSEConfig(uint32_t RCC_HSE)

    • 功能：配置外部高速振荡器（HSE）的工作模式（ON/OFF/Bypass）。

    • 参数：RCC_HSE_ON、RCC_HSE_OFF、RCC_HSE_Bypass。

  3. void RCC_PLLConfig(uint32_t RCC_PLLSource, uint32_t RCC_PLLMul)

    • 功能：配置 PLL 的时钟源和倍频系数（例如 HSE/2 × 9 = 72MHz）。

    • 参数：

      • RCC_PLLSource（选择PLL的输入源）：RCC_PLLSource_HSI_Div2、RCC_PLLSource_HSE_Div1、RCC_PLLSource_HSE_Div2

      • RCC_PLLMul（选择PLL的倍频系数）：RCC_PLLMul_2 ~ RCC_PLLMul_16（常用 RCC_PLLMul_9）。

  4. void RCC_PLLCmd(FunctionalState NewState)

    • 功能：使能或失能 PLL（锁相环）。在配置好 PLL 参数后，调用此函数启动 PLL。

    • 参数：

      • ENABLE：使能 PLL

      • DISABLE：失能 PLL

    • 典型用法：在 RCC_PLLConfig() 之后调用 RCC_PLLCmd(ENABLE)，然后等待 PLL 就绪标志（可用 RCC_GetFlagStatus(RCC_FLAG_PLLRDY) 查询）。

  5. void RCC_SYSCLKConfig(uint32_t RCC_SYSCLKSource)

    • 功能：选择系统时钟源（HSI/HSE/PLL）。

    • 参数：RCC_SYSCLKSource_HSI、RCC_SYSCLKSource_HSE、RCC_SYSCLKSource_PLLCLK。

  6. uint32_t RCC_GetSYSCLKSource(void)

    • 功能：查询当前系统时钟源。

    • 返回值：RCC_SYSCLKSource_HSI(0x00)、RCC_SYSCLKSource_HSE(0x04)、RCC_SYSCLKSource_PLLCLK(0x08)。

  7. void RCC_HCLKConfig(uint32_t RCC_SYSCLK)

    • 功能：配置 AHB 总线时钟（HCLK）的分频系数。

    • 参数：RCC_SYSCLK_Div1 ~ ````RCC_SYSCLK_Div512（常用 Div1）。

  8. void RCC_PCLK1Config(uint32_t RCC_HCLK)

    • 功能：配置 APB1 总线时钟（PCLK1）的分频系数（最高 36MHz）。

    • 参数：RCC_HCLK_Div1 ~ RCC_HCLK_Div16（常用 Div2 当 HCLK=72MHz）。

  9. void RCC_PCLK2Config(uint32_t RCC_HCLK)

    • 功能：配置 APB2 总线时钟（PCLK2）的分频系数（最高 72MHz）。

    • 参数：RCC_HCLK_Div1 ~ RCC_HCLK_Div16（常用 Div1）。

  10. FlagStatus RCC_GetFlagStatus(uint8_t RCC_FLAG)

    • 功能：查询 RCC 标志位（HSIRDY、HSERDY、PLLRDY 等）。

    • 参数：RCC_FLAG_HSIRDY、RCC_FLAG_HSERDY、RCC_FLAG_PLLRDY 等。

    • 返回：SET 或 RESET。

配置时钟树

  1. 选择时钟源（如 HSI 或 HSE）并打开它，让系统先有一个临时的、低速的时钟（确保芯片不会崩溃），等待时钟源稳定（通常需要等待 HSI/HSE 就绪标志）。

  2. 配置 PLL 的参数（分频、倍频）和分频器参数（AHB、APB1、APB2 等）。这一步只是“设置目标值”，但 PLL 此时可能尚未输出，或者输出尚未接入系统。

  3. 等待 PLL 准备好（PLL 锁定）。

  4. 通过系统时钟选择器（SW）将系统时钟切换到 PLL 输出（或 HSE，取决于设计）。

注意：在切换系统时钟之前，确保新的时钟源已经稳定并且 PLL 已经锁定，否则可能导致系统不稳定或崩溃。

配置Flash指令预取

  • Flash模块：存储程序代码和常量数据，当单片机运行时，CPU 通过总线访问 Flash 存储器来执行指令和读取数据。

  • 访问延迟：当系统时钟频率较高时，CPU 访问 Flash 存储器的速度可能跟不上，导致指令执行效率降低。

    1. <=24MHz：0个等待周期

    2. >24MHz且<=48MHz：1个等待周期

    3. >48MHz且<=72MHz：2个等待周期

    • 启用 Flash 指令预取可以提前加载指令，提高性能。

  • 在 STM32F103 系列中，当系统时钟频率超过 48MHz 时，建议启用 Flash 预取功能。

片上外设的时钟开关和复位

  1. RCC_APB2PeriphClockCmd(uint32_t RCC_APB2Periph, FunctionalState NewState)

    • 功能：使能或失能 APB2 总线上的外设时钟。

    • RCC_APB2Periph：指定 APB2 总线上的外设，例如 RCC_APB2Periph_GPIOA、RCC_APB2Periph_ADC1、RCC_APB2Periph_SPI1。

    • NewState：ENABLE（开启时钟）或 DISABLE（关闭时钟）。

  2. RCC_APB1PeriphClockCmd(uint32_t RCC_APB1Periph, FunctionalState NewState)

    • 功能：使能或失能 APB1 总线上的外设时钟。

    • RCC_APB1Periph：指定 APB1 总线上的外设，例如 RCC_APB1Periph_TIM2、RCC_APB1Periph_I2C1、RCC_APB1Periph_USART2。

    • NewState：ENABLE（开启时钟）或 DISABLE（关闭时钟）。

  3. RCC_AHBPeriphClockCmd(uint32_t RCC_AHBPeriph, FunctionalState NewState)

    • 功能：使能或失能 AHB 总线上的外设时钟。

    • RCC_AHBPeriph：指定 AHB 总线上的外设，例如 RCC_AHBPeriph_DMA1、RCC_AHBPeriph_CRC、RCC_AHBPeriph_FSMC。

    • NewState：ENABLE（开启时钟）或 DISABLE（关闭时钟）。

  4. RCC_APB2PeriphResetCmd(uint32_t RCC_APB2Periph, FunctionalState NewState)

    • 功能：复位 APB2 总线上的外设。

    • RCC_APB2Periph：指定 APB2 总线上的外设（同上 RCC_APB2PeriphClockCmd 的参数）。

    • NewState：ENABLE（将外设置于复位状态）或 DISABLE（释放复位）。

  5. RCC_APB1PeriphResetCmd(uint32_t RCC_APB1Periph, FunctionalState NewState)

    • 功能：复位 APB1 总线上的外设。

    • RCC_APB1Periph：指定 APB1 总线上的外设（同上 RCC_APB1PeriphClockCmd 的参数）。

    • NewState：ENABLE（将外设置于复位状态）或 DISABLE（释放复位）。
